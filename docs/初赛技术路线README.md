这份文档只服务当前比赛主线：

- 赛题：挂点金具螺栓缺失
- 阶段：初赛
- 当前日期：2026-03-24
- 截止日期：2026-03-31

目标不是继续发散研究，而是在剩余 7 天内，把现有资产收束成一个可提交、可运行、可解释、可复现的初赛成果包。

规范依据：

> `大赛需求` 明确写明“初赛（3月31号截止）”。

## 2. 规则理解

当前本地规范存在两层口径，必须按更严格的一层执行。

### 2.1 名义提交口径

按 `docs/规范_md/大赛需求/大赛需求.md` 与 `docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`，初赛名义上需要提交三类成果：

1. 生图模型
2. 高质量数据集
3. 高质量数据集说明文档

规范依据：

> `大赛需求`：初赛需提交“生图模型”“高质量数据集”“高质量数据集说明文档”。  
> `提交格式说明`：第一部分“提交内容”再次明确三类核心成果物。  
> 来源：`docs/规范_md/大赛需求/大赛需求.md`，`docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`

### 2.2 实际评分口径

按 `docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`，初赛更像是下面这条链路：

1. 缺陷识别模型可多次提交，系统自动评分，取最高分
2. 初赛结束前 7 天，专家仅对检测分前 60 名队伍的最终生图模型统一生成 300 张图并评审
3. 初赛总分 = 缺陷识别模型得分 + 生图模型得分
4. 前 30 名晋级

规范依据：

> `操作指南`“赛程介绍”写明：  
> 1）缺陷识别算法模型可多次上传，自动计算并取最高分；  
> 2）初赛结束前 7 天，仅对检测分前 60 名队伍的最终文生图/图生图模型统一生成 300 张图后专家评审；  
> 3）初赛得分 = 缺陷识别算法模型得分 + 文生图/图生图模型得分；  
> 4）前 30 名晋级复赛。  
> 来源：`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`

### 2.3 当前执行原则

从今天开始，不能再把初赛理解为“只要交齐三件套即可”。

当前最保守、最安全的理解是：

- 检测模型在初赛是实质必做项
- 生图模型不能为零，但不应再作为唯一主线
- 数据集质量、说明文档完整性、提交物合规性必须同步推进

规范依据：

> `大赛需求` 与 `提交格式说明` 只列出三件套提交物；  
> 但 `操作指南` 给出了更强约束的实际评分链路，因此执行时必须按“更严格并集”理解，而不能只按名义提交物理解。  
> 来源：`docs/规范_md/大赛需求/大赛需求.md`，`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`，`docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`

## 3. 当前现状判断

### 3.1 已有资产

- 100 张挂点金具螺栓缺失真实缺陷图
- 对应框标注
- 一部分 SAM2 mask 资产
- SDXL 局部修复原型与服务器推理能力
- 仓库已按 `bolt/` 主线拆分

### 3.2 当前短板

- 没有稳定的健康图闭环
- 没有成熟、稳定、可批量验收的生图模型主链
- 检测 baseline 尚未形成“提交级”闭环
- 数据集说明文档还未冻结成最终版
- 比赛打包物仍未成形

### 3.3 结论

现在最缺的不是新算法，而是提交级资产冻结。

## 4. 技术路线总判断

当前不再采用“双向生成 + 大规模参数搜索 + 完整健康图闭环”的扩张路线。

当前推荐路线为：

`检测先卡位 + 真实数据集冻结 + 生图最小可运行包`

这是一个资产优先型初赛路线，而不是研究优先型路线。

## 5. 总体技术路线

整体拆成四条并行但强耦合的工作线。

### 5.1 工作线 A：检测保底线

目标：

- 尽快拿到可提交、可反复刷新最高分的缺陷识别 baseline
- 在初赛评分链路里先占住位置

规范依据：

> `操作指南` 明确把“缺陷识别模型精确率、召回率”列为赛题专项指标，并说明初赛检测模型分可多次提交取最高分。  
> 来源：`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`

策略：

- 只做单类 `missing_fastener`
- 使用最轻 baseline
- 不做检测框架横跳
- 先完成一次真实训练、验证、推理、结果导出

交付物：

- 可运行的检测训练脚本
- 最小可用模型权重
- 一次真实评估结果
- 可重复提交的推理入口

仓库落点：

- `bolt/detect/`
- `bolt/detect/scripts/`

### 5.2 工作线 B：真实数据集冻结线

目标：

- 把现有 100 张真实缺陷图冻结成提交级高质量数据集主体

策略：

- 以真实图为主，生成图为辅
- 清洗歧义样本
- 统一类别、命名、标注格式
- 做 train/val/test 防泄漏切分
- 产出统计表、样本分布、异常样本记录

特别注意：

- 官方标注口径仍然是矩形框，不是 mask
- SAM2 mask 是资产增强与质检辅助，不是官方标注替代物
- 要严格核查“本身就没有螺母”“因视角无法判断”的样本

规范依据：

> `标注说明文档` 明确写明“本次数据标注仅支持矩形框标注”，并列出“本身就没有螺母”“视角或遮挡无法判断不标”的例外规则。  
> 来源：`docs/规范_md/输电线路巡检挂点金具螺栓缺失数据集标注说明文档/auto/输电线路巡检挂点金具螺栓缺失数据集标注说明文档.md`

交付物：

- 冻结版图片集
- 对应标注文件
- 数据集 manifest
- 标签统计和 split 统计

仓库落点：

- `bolt/dataset/`
- `bolt/docs/dataset_contract.md`

### 5.3 工作线 C：生图最小可运行线

目标：

- 交出一个可运行、可复现、可说明的生图模型提交物
- 不再追求“最好看的生成效果”

策略：

- 停掉 `缺陷 -> 健康`
- 停掉双向路线
- 只保留一条最窄的 `图生图` 管线
- 只做“同类缺陷变体生成”
- 所有生成样本必须人工复核后才能进入最终数据集

当前约束：

- 没有健康图，不能再押完整健康图闭环
- 不再继续大规模 ROI / prompt / mask 参数搜索
- 不把 SAM2 全量资产化作为初赛主胜负手

规范依据：

> `提交格式说明` 对生图模型提交物要求的是“可正常运行的模型文件及配套代码、环境依赖、镜像信息等”，并要求支持昇腾 910B 运行说明；  
> `操作指南` 对生图模型的真正评分更强调“生成结果的可用张数”，不是主观观感打磨。  
> 来源：`docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`，`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`

交付物：

- `infer.py`
- `requirements.txt`
- `README.md`
- 运行手册
- 910B 适配说明或镜像信息
- 一小批经过人工筛选的代表性生成样例

仓库落点：

- `bolt/generate/`
- `bolt/generate/scripts/`
- `bolt/package/`

### 5.4 工作线 D：说明文档与打包线

目标：

- 把前面三条线的结果收束成比赛能接收的提交包

策略：

- 立即构造本地 build 目录
- 按提交格式说明搭建目录骨架
- 文档先写骨架，再逐项填充
- 提前做命名、体积、结构、可运行性检查

文档必须覆盖：

- 基本信息完整性
- 内容特征完整性
- 建设过程完整性
- 应用说明完整性
- 局限性说明
- 安全边界说明

规范依据：

> `提交格式说明` 附件 1 已经把高质量数据集说明文档结构写死，包括：  
> 基本信息完整性、内容特征完整性、建设过程完整性、应用说明完整性，以及其中的数据规模、格式规范、文件结构、访问渠道、特殊业务规则、局限性说明、版本控制记录、安全规范、典型应用案例等栏目。  
> 来源：`docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`

仓库落点：

- `bolt/package/`
- `docs/`

## 6. 明确砍掉的方向

以下内容从现在起不再作为主投入方向。

1. 大规模 SDXL 参数搜索
2. 双向生成路线
3. 全量 SAM2 mask 资产化
4. 健康图完整闭环执念
5. legacy demo 路线
6. 检测框架横向反复比较

原因只有一个：这些内容很难在 7 天内直接提升初赛提交质量和晋级概率。

规范依据：

> 当前离截止仅剩 7 天；而 `操作指南` 已把初赛得分机制绑定到“检测模型得分 + 生图模型得分”上，所以任何不能直接提升检测可提交性、生成可用张数、数据质量与提交合规性的工作，都属于低收益分支。  
> 来源：`docs/规范_md/大赛需求/大赛需求.md`，`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`

## 7. 当前最合理的主线结构

### 7.1 主线目标

在 `2026-03-31` 前，完成一个“真实数据主体清晰、检测可提交、生图可运行、文档完整、打包规范”的初赛成果包。

规范依据：

> 这五个词分别对应规范中的核心约束：  
> `真实数据主体清晰` 对应高质量数据集与标注质量；  
> `检测可提交` 对应初赛实际评分门槛；  
> `生图可运行` 对应生图模型提交要求；  
> `文档完整` 对应说明文档模板；  
> `打包规范` 对应压缩格式、命名规则、目录结构、910B 适配说明。  
> 来源：`docs/规范_md/大赛需求/大赛需求.md`，`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`，`docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`

### 7.2 主线优先级

1. 检测 baseline 起跑并拿到结果
2. 100 张真实数据集冻结
3. 高质量数据集说明文档成稿
4. 生图模型最小可运行包成形
5. 少量高置信生成样本补充

### 7.3 主线原则

- 先保底，再冲分
- 先交付，再优化
- 先真实，再合成
- 先合规，再漂亮

## 10. 当前的 Done Definition

只有同时满足下面条件，才算初赛路线完成。

1. 有一个可提交的检测 baseline
2. 有一个冻结版高质量数据集
3. 有一份完整的数据集说明文档
4. 有一个最小可运行的生图模型包
5. 有清晰的目录结构、命名规则、运行说明和依赖说明
6. 所有最终入库生成样本都经过人工复核

规范依据：

> 其中 1）来自初赛实际评分口径；  
> 2）3）来自初赛名义提交物；  
> 4）5）来自生图模型提交格式要求；  
> 6）来自高质量数据集“准确无错误、完整无缺失、模型应用支撑性”的基本要求。  
> 来源：`docs/规范_md/大赛需求/大赛需求.md`，`docs/规范_md/广东高质量数据集创新大赛-能源赛题-操作指南/auto/广东高质量数据集创新大赛-能源赛题-操作指南.md`，`docs/规范_md/高质量数据集创新大赛－能源赛道成果物提交格式说明/auto/高质量数据集创新大赛－能源赛道成果物提交格式说明.md`

## 11. 当前一句话策略

不要再赌“把 SDXL 再磨漂亮一点就能翻盘”。

`真实数据质量 + 检测先卡位 + 生图最小可运行 + 文档零硬伤`
