# 数据集人工复核清单

适用范围：`missing_fastener` 检测主线的数据整理与框标注复核。目标是用最少的人力，先挡住会影响训练质量或隐私边界的明显问题。

## 图像质量

- [ ] 图像能看清挂点金具区域，主体没有严重失焦到无法判断。
- [ ] 没有因裁切、旋转、重编码导致关键缺陷区域丢失。
- [ ] 同一原图的明显重复件、相邻帧近重复件已标记为同组，避免后续 split 泄漏。
- [ ] 图像中若存在极端遮挡、强反光、过曝、欠曝、雨雾影响，已在备注里标明。

## 标注质量

- [ ] 当前样本的类别语义正确，正类统一认定为 `missing_fastener`。
- [ ] 每个缺失实例单独成框，没有把多个目标合成一个大框。
- [ ] 框覆盖的是缺失证据区域，没有大量吞入无关背景。
- [ ] 框坐标落在图像范围内，没有负值、越界或空框。
- [ ] 空图或健康候选图没有误留正类框。
- [ ] 模糊难例、边界不清样本已标 `needs_fix` 或写入备注，而不是直接混入已通过样本。

## 元数据与划分

- [ ] `sample_id`、`scene_id`、`capture_group_id` 已填写，且能支撑组级切分。
- [ ] `is_positive`、`bbox_count`、`review_status` 一致，不存在自相矛盾。
- [ ] train/val/test 划分按场景或拍摄组隔离，没有把同组样本拆到多个 split。
- [ ] 原图裁切、重采样图、轻微增强副本没有跨 split。

## 隐私与入库边界

- [ ] 当前复核材料不包含需要提交到 Git 的真实图像、真实标注或真实统计导出。
- [ ] 文档、脚本示例、截图里没有泄露真实路径、设备编号、采集批次或其他敏感信息。
- [ ] 若样本含不确定的敏感信息，已标记并留在本地私有区处理，没有写入仓库文档。

## 复核结论

建议只保留四种结论，避免状态膨胀：

- `approved`：可进入 detector 导出
- `needs_fix`：可以修后再入集
- `rejected`：不应进入当前数据集
- `pending`：尚未完成判断
