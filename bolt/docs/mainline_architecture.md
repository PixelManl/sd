# 螺栓缺失三线架构

## 目标

围绕 `挂点金具螺栓缺失` 建立一条明确主线和两条支撑线，避免把检测、健康图、SAM2 做成三套互相竞争的方向。

## 一条主线

### 检测主线

主线任务是：

- 把现有框标注整理成高质量数据集
- 拉起单类 `missing_fastener` 检测 baseline
- 形成可训练、可验证、可推理、可提交的识别闭环

为什么它是主线：

- 比赛最终会落到识别模型
- 官方监督格式本来就是矩形框
- 现有样本最直接支持这条路径

检测主线的输出：

- 统一后的图像与框标注
- train/val/test 划分
- baseline 模型与评估指标
- 误检/漏检样例集
- 推理与打包脚本

## 两条支撑线

### 健康数据线

健康数据线不是为了“多收点图”，而是为了三件事：

- 提供正常状态先验
- 提供负样本和 hard negative
- 提供后续缺陷注入所需的母图

它当前最应该产出的不是 mask，而是：

- 小规模高匹配健康图种子集
- 候选框与人工确认结果
- 来源记录和筛选清单

健康数据线的作用是支撑检测和未来生成，不应该独立膨胀成新的主线。

### SAM2 资产线

SAM2 线不是比赛官方标注线，而是内部增强资产线。

它的任务是把局部结构做成可复用资产：

- `core_mask`：紧贴可见缺陷证据
- `edit_mask`：略大一圈的局部编辑区域
- `overlay`
- `metadata`
- `lineage`

它的价值在于：

- 为局部编辑和缺陷注入做准备
- 提供更细的 ROI 审核视角
- 为后续增强和难例构造提供结构先验

## 三线交叉

### 检测 -> 健康数据

检测线告诉健康数据线：

- 哪些背景最容易误报
- 哪些视角最有价值
- 哪些正常样本最值得收

### 健康数据 -> 检测

健康数据线回馈检测线：

- 真实负样本
- hard negative
- 后续合成正样本的母图

### 检测 -> SAM2

检测线告诉 SAM2：

- 哪些实例最值得做精细资产
- 哪些 ROI 是高价值局部结构

### SAM2 -> 检测

SAM2 回馈检测线：

- 更细的 ROI 审核材料
- 后续受控增强素材
- 结构难例构造依据

### 健康数据 -> SAM2

健康图为 SAM2 提供完整结构的正常样本。

### SAM2 -> 健康数据

SAM2 可以把健康图沉淀成可复用的局部结构资产，而不只是整图库存。

## 当前最高收益动作

1. 先把缺陷图和框标注整理成检测可用的数据集
2. 立即跑一个轻量单类 detector baseline
3. 同步建立小规模健康图种子集和筛选清单
4. 同步做一批代表性 SAM2 pilot，而不是一开始全量铺开

## 反模式

- 不要等健康图齐了再启动检测
- 不要把 SAM2 当成官方标注替代
- 不要在 baseline 还没站住前，先把精力砸到大规模生图
- 不要把“收健康图”做成脱离主线的独立收藏任务
